# ZZcare

수면과 코골이를 자동으로 기록하고 분석해주는 수면 헬스케어 서비스입니다.

Health Connect에서 가져온 실제 수면 데이터와, AI 모델이 자동으로 감지·분류한 코골이 데이터를 결합해서 사용자에게 매일 밤의 수면 리포트를 제공합니다.

---

## 서비스가 하는 일

1. **수면 기록**: Android Health Connect(삼성 헬스 등)와 연동해서 취침/기상 시각, 수면 단계(깊은잠·REM·얕은잠·깸)를 자동으로 가져옵니다.
2. **코골이 자동 감지**: 수면 중 마이크로 수집한 소리를 AI 모델에 넣어 자동으로 분석합니다.
   - **1단계 분류**: 코골이 소리인지 아닌지 구분
   - **2단계 분류**: 코골이가 아니라면 그 소리가 정확히 무엇인지 세부 라벨링 (예: 빗소리 등 생활 소음)
   - 이 결과로 코골이 시간, 강도(dB), 발생 횟수, 시간대별 패턴을 자동으로 산출합니다.
3. **리포트 제공**: 위 두 데이터를 하루 단위로 합쳐서 수면 점수, 수면 단계, 코골이 통계, 일별/월별 추이를 사용자에게 보여줍니다.
4. **로그인/계정 관리**: 카카오 로그인으로 간편하게 가입하고, 서버에 사용자 데이터를 저장해 기록을 이어갈 수 있습니다.

즉, **"사용자가 직접 아무것도 입력하지 않아도, 자는 동안 자동으로 수면과 코골이가 기록되는 것"**이 이 서비스의 핵심입니다.

---

## 전체 구성

서비스는 크게 세 부분으로 이루어져 있습니다.

### 📱 모바일 앱 (Flutter)

사용자가 실제로 사용하는 화면입니다. 수면 / 코골이 / 통계 3개 탭으로 구성되어 있고, Health Connect 동기화와 AI 코골이 측정을 실행하는 클라이언트 역할을 합니다.

- **수면 탭**: 지난밤 수면 점수, 수면 단계 비율, 취침·기상 시각, 목표 대비 부족 수면
- **코골이 탭**: 총 코골이 시간, 평균/최대 dB, 발생 횟수, 시간대별 강도 그래프
- **통계 탭**: 일별/월별 추이를 수면 점수·수면 시간·소음·부족 수면 기준으로 차트화

### 🧠 AI 코골이 분류 모델

마이크로 수집한 오디오를 분석해서 코골이 여부와 종류를 자동으로 판별하는 모델입니다.

- 코골이 / 비코골이를 우선 구분하고, 비코골이 소리는 각각 무엇인지(예: 빗소리) 세부적으로 라벨링하도록 학습되었습니다.
- 이 판별 결과가 코골이 시간·dB·발생 횟수 등 정량적인 지표로 변환되어 앱에 표시됩니다.
- _(모델 아키텍처와 추론 위치(온디바이스/서버)는 추후 확정되는 대로 이 문서에 채워 넣을 예정입니다.)_

### ☁️ 백엔드 (FastAPI + MongoDB)

- 카카오 로그인 정보를 받아 사용자 계정을 저장/조회
- 사용자별 수면 기록 관리의 기반이 되는 서버

---

## 설계 원칙

- **목업 데이터 없음**: 화면에 보이는 모든 값은 Health Connect 또는 AI 코골이 측정에서 나온 실제 데이터입니다.
- **실패해도 안 막히는 화면**: 데이터를 못 가져와도 앱은 정상적으로 뜨고 0으로 표시되며, 상단 배너로만 조용히 상태를 안내합니다.
- **수면 점수는 룰 기반 계산**: 목표 대비 수면시간 차이, 깊은잠/REM 비율, 깨어있던 비율을 기준으로 100점에서 감점하는 방식입니다. 의학적으로 검증된 공식이 아니라 서비스용 휴리스틱입니다.

## 기술 스택

| 영역             | 내용                                                |
| ---------------- | --------------------------------------------------- |
| 모바일 앱        | Flutter (Dart)                                      |
| 수면 데이터 연동 | `health` 패키지 (Android Health Connect)            |
| 코골이 분석      | AI 오디오 분류 모델 (코골이 / 비코골이 세부 라벨링) |
| 차트/시각화      | `fl_chart`                                          |
| 로그인           | 카카오 로그인 SDK                                   |
| 백엔드           | FastAPI                                             |
| 데이터베이스     | MongoDB                                             |

---

## 알려진 제약

- AI 코골이 분류 모델의 구체적인 아키텍처와 추론 위치(온디바이스 vs 서버)는 아직 문서에 반영되지 않음
- 코골이 세부 라벨 종류(빗소리 외 다른 카테고리)는 추가 확인 필요
- 통계 탭의 월별 보기는 아직 실제 데이터 집계 로직이 없음 (현재는 최근 기록을 월별로 묶어 평균 내는 작업이 남아있음)
- 수면 점수 계산식은 의학적 검증을 거치지 않은 서비스용 휴리스틱
